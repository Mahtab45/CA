How Does Internet Work

Principle 1: The Need to Share Information

  • We need to move information from one computer to another, even if they’re far apart. The simplest way is carrying it physically (like with a USB), but that’s slow. A better way is to connect the computers directly using a cable (ethernet/fiber). By sending signals (1s and 0s) through the cable, we create the most basic network — a direct link for communication.

Principle 2: The Problem of Scale

  • Directly connecting every computer to every other one doesn’t scale (too many cables). The solution is a central device (Switch) where all computers connect. The switch forwards messages only to the intended computer. This creates a Local Network (LAN) for efficient communication.

Principle 3: The Problem of Connecting Networks

  • Local networks alone can’t scale to connect with every computer in other offices.
  • The solution is a Router, which links one network to another and forwards traffic.
  • By connecting routers together, multiple networks can interconnect.
  • The Internet is essentially a huge network of networks joined through routers.

Principle 4: The Problem of Finding Anyone (Addressing)

  • In a global network, every device needs a unique identifier to be found.
  • This led to the creation of the IP Address, similar to a postal address.
  • Each device gets a unique IP (e.g., 142.250.184.142).
  • Routers read the destination IP and forward the message one step closer, using the best next hop.

Principle 5: The Problem of Reliability and Fairness

  • Sending large files at once is unfair (blocks others) and unreliable (corruption ruins the file).
  • Solution: break data into small packets with payload, source/destination IPs, and sequence numbers.
  • Packets may travel different routes; the receiver reassembles them and requests missing ones.
  • These rules form a Protocol, mainly TCP/IP, which ensures reliable and fair data transfer.

IPv4(Internet Protocol version 4)

What is IPv4?

IPv4(Internet Protocol version 4) is the classic Internet Protocol address format. It is a 32-bit number written as four decimal numbers (0–255) separated by dots:

Example: 172.217.16.142

Why this matters

  • 32 bits → 2^32 ≈ 4.3 billion total addresses.
  • At first that seemed enough, but now laptops, phones, and IoT devices use them up fast.
  • This shortage led to public vs private IPs and IPv6.

Key Takeaway

IPv4 gives every device a unique address, but 32 bits are not enough for today’s Internet — so we needed NAT, private IPs, and IPv6.

IPv6(Internet Protocol version 4)

Why IPv6?

IPv6 was created to solve the address exhaustion problem of IPv4.

Address Size Difference

  • IPv4: 32-bit → about 4.3 billion addresses
  • IPv6: 128-bit → a huge leap in available addresses

The number of possible IPv6 addresses is 2^128, which is roughly:

340,000,000,000,000,000,000,000,000,000,000,000,000 (340 undecillion)

IPv6 Address Format

Because IPv6 addresses are long, they use hexadecimal (0–9, a–f) and are written in a colon-separated format.

Example:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

  • Eight groups of four hexadecimal characters
  • Groups separated by colons (:)

MAC Address

What is a MAC Address?

A MAC (Media Access Control) Address is a unique, permanent serial number burned into every network-capable piece of hardware (your laptop's Wi-Fi card, your phone, your smart TV, the network port on your desktop).

Format

Written in hexadecimal, usually 12 characters long:

  • Colon-Separated: 3C:22:FB:A3:B4:C5
  • Hyphen-Separated: 3C-22-FB-A3-B4-C5
  • Period-Separated: 3c22.fba3.b4c5
  • No Separators: 3C22FBA3B4C5

How to Find MAC Address

Windows:

  1. Open Command Prompt
  2. Run command: ipconfig /all
  3. Or use: getmac /v (to see only MAC addresses).

macOS:

  1. Open Terminal
  2. Run command: ifconfig

Port Number

  • Port numbers are numerical identifiers (0–65535) used in networking to identify specific applications or services running on a device.
  • A port number is a 16-bit unsigned integer. so valid range is 0 to 65535.
    • 16-bit: This means the number is stored using 16 binary digits (1s and 0s).
    • Unsigned: This means it cannot be negative. The values are all positive.
  • Relation with IP:
    • IP Address → Identifies the machine (like ghar ka address).
    • Port Number → Identifies the service/application on that machine (like ghar ke andar room number).

Categories of Port Numbers

  1. Well-Known Ports (0–1023):
    • Reserved for common services.
    • Example :
      • HTTP → 80
      • HTTPS → 443
      • FTP → 21
      • SSH → 22

  2. Registered Ports (1024–49151):
    • Used by software/applications that are not as universal as well-known ports.Companies and developers can register a port for their software to avoid conflicts with other applications.
    • Example :
      • MySQL → 3306
      • PostgreSQL → 5432

  3. Dynamic / Private Ports (49152–65535):
    • Temporary ports assigned by the operating

DNS(DOMAIN NAME SYSTEM)

  • DNS is a global, distributed system that translates human-friendly domain names (like google.com) into computer-friendly IP addresses (like 142.250.184.142).

Why we need DNS?

  • Humans remember names easily, not numbers.
  • Computers communicate using IP addresses.
  • DNS works like the phonebook of the Internet, mapping names to numbers.
